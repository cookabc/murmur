# Voice Input for macOS 逆向产品文档

## 1. 文档说明

本文档基于当前仓库实现逆向整理，目标是描述这个项目在现阶段已经实现的产品能力、交互流程、依赖边界和待完善范围。

文档依据主要来自以下位置：

- `README.md`
- `dist/index.html`
- `dist/app.js`
- `src-tauri/src/*.rs`
- `src-tauri/tauri.conf.json`

本文档描述的是“当前真实产品”，不是理想规划版本。凡是 README 中提到、但源码未完整落地的能力，均会单独标注。

## 2. 产品定义

### 2.1 产品名称

Voice Input for macOS

### 2.2 产品定位

一个运行在 macOS 上、像输入法一样工作的本地语音输入工具。用户通过全局唤起动作开始录音，录音结束后使用本地 ASR 工具进行转写，并将文本插入当前输入位置。

### 2.3 产品目标

- 为 macOS 用户提供低摩擦的语音转文字输入能力
- 尽可能保持本地处理，避免上传音频到云端
- 支持像输入法一样跨应用输入，而不是只在应用内查看转写结果
- 允许用户查看最近转写历史并重复使用

### 2.4 核心价值

- 隐私优先：音频和转写流程本地完成
- 跨应用输入：输出目标不是本应用，而是当前激活应用
- 操作简单：围绕“开始录音 / 停止录音 / 自动粘贴”建立最短路径

## 3. 目标用户

### 3.1 核心用户

- 需要频繁输入短文本的 macOS 用户
- 对隐私较敏感，不希望语音数据经过云端服务的用户
- 希望在聊天、写作、笔记、邮件等多个应用中复用语音输入能力的用户

### 3.2 典型使用场景

- 在 IM 工具中快速输入一段回复
- 在笔记或文档应用中口述短段落
- 在 IDE、终端旁边完成自然语言备注输入
- 在不方便长时间键盘输入时以语音代替文本输入

## 4. 当前功能范围

### 4.1 已实现功能

#### A. 音频录制

- 通过 `ffmpeg` 调用 macOS `avfoundation` 录制默认输入设备音频
- 输出为单声道、16kHz 的 WAV 文件
- 单次录音上限为 5 分钟

#### B. 本地语音识别

- 调用 `coli asr` 命令进行离线识别
- 支持模型选项：`sensevoice`、`whisper`
- 可返回识别文本、语言信息、音频时长

#### C. 转写结果处理

- 在应用内展示转写结果
- 支持复制转写结果
- 支持将转写结果粘贴到当前激活应用
- 支持根据设置自动粘贴

#### D. 历史记录

- 本地保存历史记录
- 支持查看、复制、粘贴、删除单条历史记录
- 支持清空历史记录
- 支持限制历史记录保留条数

#### E. 设置项

- 默认热键字符串显示
- ASR 模型选择
- AI Polish 开关
- Auto-paste 开关
- Paste method 开关
- History Count 设置

### 4.2 部分实现或与宣称不一致的功能

#### A. 全局热键

产品层面将其定义为核心入口，但当前后端只保存热键字符串，没有完成实际动态注册逻辑。

现状结论：

- “热键作为概念存在”
- “热键修改 UI 存在”
- “热键真正生效与动态切换未完整落地”

#### B. AI Polish

界面和设置中暴露了该能力，但后端注释明确说明 `coli asr` 当前并不支持这个参数，因此现在更接近“预留开关”而非稳定功能。

#### C. 输入法式交互形态

产品目标应是“输入法式的全局唤起和文本插入”，但当前实现仍以普通主窗口为核心。托盘图标存在，然而后台常驻、无窗口主流程和真正的全局输入体验仍未完整落地。

### 4.3 未实现能力

- 热键录制规则的实时修改与持久化注册
- 真正可验证的输入法式全局唤起交互
- 多语言 UI / 国际化
- 启动自检与依赖缺失引导
- 后台服务模式或 headless 模式
- 录音输入源切换、采样率配置等高级设置

## 5. 典型用户流程

### 5.1 主流程

1. 用户启动应用。
2. 应用加载本地设置与历史记录。
3. 用户点击 “Start Recording” 或通过外部触发开始录音。
4. 应用切换到录音中状态，显示计时与波形动画。
5. 用户点击 “Stop Recording” 或再次触发停止录音。
6. 应用调用本地 ASR 命令对录音文件进行转写。
7. 应用展示转写结果。
8. 若启用了自动粘贴，则将文本粘贴到当前应用。
9. 同时写入历史记录，供后续复制或再次粘贴。

### 5.2 历史复用流程

1. 用户切换到 History 标签页。
2. 查看近期转写结果。
3. 对指定记录执行 Copy / Paste / Delete。
4. 如有需要，可一键清空历史。

### 5.3 设置流程

1. 用户在 Settings 页面修改模型、自动粘贴等参数。
2. 点击保存。
3. 设置写入本地 JSON 存储。
4. 后续录音 / 转写流程读取这些设置作为默认行为。

## 6. 信息架构与界面结构

当前主界面包含三块核心区域：

### 6.1 Recording Section

- 麦克风图标
- 录音波形动画
- 状态文案
- 录音计时
- Start / Stop 按钮

### 6.2 Result Section

- 转写文本展示
- 复制按钮
- Paste 按钮
- New Recording 按钮

### 6.3 Tabs Section

- Settings 标签页
- History 标签页

这说明当前产品仍偏向“小型控制面板”，还没有完全过渡到输入法式工具形态。

## 7. 设置项定义

当前持久化设置结构如下：

| 字段 | 默认值 | 说明 |
| --- | --- | --- |
| `hotkey` | `Cmd+Shift+V` | 热键显示与存储值 |
| `model` | `sensevoice` | ASR 模型 |
| `polish` | `true` | 预留的文本润色开关 |
| `auto_paste` | `true` | 转写完成后是否自动粘贴 |
| `use_applescript` | `true` | 是否使用 AppleScript 方案粘贴 |
| `history_count` | `50` | 历史记录最大保留数 |

## 8. 数据与存储

### 8.1 设置与历史存储

- 路径：`~/.voice-input-mac/settings.json`
- 内容：设置对象 + 历史数组
- 存储方式：JSON 文件

### 8.2 历史记录字段

- `id`
- `text`
- `timestamp`
- `lang`
- `duration`

## 9. 依赖与运行前提

### 9.1 系统要求

- macOS 13.0+

### 9.2 外部依赖

- `ffmpeg`：负责录音
- `coli`：负责本地 ASR
- Rust / Cargo：开发与构建
- Node.js：Tauri 前端开发流程

### 9.3 权限依赖

- 麦克风权限
- 辅助功能权限或脚本控制权限（用于模拟粘贴）

## 10. 当前产品边界

### 10.1 明确成立的产品能力

- 应用内可以完成录音、转写、结果查看、历史存储
- 可以通过命令行工具链实现本地离线语音转文字
- 可以尝试将结果插入当前应用

### 10.2 不能当作稳定承诺的能力

- 动态修改全局热键后立即生效
- 真正意义上的“输入法式全局唤起、无主窗口主操作”
- AI Polish 对文本质量的稳定提升
- 对所有特殊字符、权限场景和外部依赖缺失的健壮处理

## 11. 当前版本更适合的对外定义

相比“完整的本地语音输入法式工具”，当前版本更准确的描述应为：

“一个面向 macOS 的本地语音转写与跨应用插入原型，已经具备可运行的核心链路，但在热键、常驻形态和稳定性方面仍需完善。”

## 12. 后续产品演进建议

### 阶段一：把主链路做稳

- 确保录音开始 / 停止可靠
- 确保转写调用不会冻结界面
- 确保粘贴策略在权限失败时可回退

### 阶段二：兑现产品承诺

- 完成全局热键注册与热键修改
- 完整实现全局唤起、托盘辅助入口和后台常驻模式
- 补齐启动自检与缺失依赖引导

### 阶段三：增强可用性

- 输入源选择
- 热键冲突检测
- 国际化
- 历史搜索、收藏、再次编辑
